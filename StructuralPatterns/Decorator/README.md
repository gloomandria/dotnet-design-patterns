# Decorator Pattern

## Description
Le pattern Decorator permet d'ajouter dynamiquement de nouvelles fonctionnalités à un objet en l'enveloppant dans un objet décorateur.

## Utilisation
- Ajouter des responsabilités à des objets de manière dynamique
- Alternative flexible à l'héritage
- Streams en .NET (FileStream, BufferedStream, etc.)

## Avantages
- Plus flexible que l'héritage
- Principe de responsabilité unique
- Principe ouvert/fermé
- Combinaison de comportements à l'exécution

## Inconvénients
- Beaucoup de petites classes
- Difficile à déboguer (empilement de décorateurs)
- Configuration complexe
