# Facade Pattern

## Description
Le pattern Facade fournit une interface unifiée et simplifiée à un ensemble complexe de sous-systèmes.

## Utilisation
- Simplifier l'utilisation d'une bibliothèque complexe
- Découpler le code client des sous-systèmes
- API publique d'une bibliothèque

## Avantages
- Simplifie l'interface client
- Découple le code du sous-système
- Facile à utiliser

## Inconvénients
- Peut devenir un objet god (anti-pattern)
- Peut cacher trop de fonctionnalités
