# Adapter Pattern

## Description
Le pattern Adapter permet à des interfaces incompatibles de travailler ensemble. Il convertit l'interface d'une classe en une autre interface attendue par le client.

## Utilisation
- Intégration de bibliothèques tierces
- Réutilisation de code existant avec une interface différente
- Migration progressive vers une nouvelle API

## Avantages
- Principe de responsabilité unique
- Principe ouvert/fermé
- Permet la réutilisation de code existant

## Inconvénients
- Complexité accrue du code
- Peut ajouter une couche d'abstraction supplémentaire
